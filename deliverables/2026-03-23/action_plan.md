# Top Opportunity: AI Knowledge Bot Platform

**Kategorie:** Bot Platform · **Final Score:** 2033.9 · **Builder Fit:** 3/5 · **Schwierigkeit:** 2/5

## Zusammenfassung

**Problem:** Teams beantworten dieselben internen Fragen immer wieder — Wissen ist in Docs verstreut

**Zielkunde:** HR-Leads, Support-Teamleiter, Ops-Manager in 10-100 Personen Unternehmen

**Lösung:** Spezialisierter Slack/Discord/WhatsApp-Bot der Unternehmensdokumente kennt und Team-Fragen beantwortet. Setup: Docs verlinken, Bot einladen, fertig.

**Warum jetzt:** The Bing AI bot has been secretly running GPT-4 (HackerNews AI); How Descript enables multilingual video dubbing at scale (OpenAI Blog); LeRobot v0.5.0: Scaling Every Dimension (Hugging Face Blog)

**Revenue-Modell:** subscription · **MVP in:** 2 Wochen

## 1-Tages-Plan

### Block 1: Research (2h)
- Baue einen minimalen Slack-Bot der auf /frage reagiert und eine FAQ aus Markdown beantwortet
- Analysiere die Top-3-Konkurrenten und ihre Schwächen
- Dokumentiere: Was fehlt am Markt?

### Block 2: Validation (2h)
- Füge RAG über Team-Dokumente hinzu (Markdown/PDF-Ingestion)
- Schreibe 3 potentiellen Nutzern eine kurze Nachricht
- Sammle initiales Feedback

### Block 3: Build Spike (4h)
- Teste mit einem echten Team (5 Personen), sammle Feedback, füge Admin-Commands hinzu
- Teste den Kern-Workflow end-to-end
- Dokumentiere technische Risiken

## 3-Tages-Plan

- **Tag 1:** Research + Validation + erster Prototyp
- **Tag 2:** Core MVP implementieren, Kernfeature funktional
- **Tag 3:** Polish, Landing Page, erste Distribution (HN Show, PH, Reddit)

## MVP Scope

**In Scope:**
- Kernfunktion die das Hauptproblem löst
- Minimales UI (kann CLI oder simple Web-App sein)
- Ein klarer Workflow von Input → Output
- Basis-Auth und Nutzerverwaltung falls SaaS

**Anti-Scope (NICHT im MVP):**
- Team-Features, Rollen, Permissions
- Mehr als 1 Integration
- Mobile App
- Analytics Dashboard
- Eigene Modelle trainieren

## Validierungs-Skript

**Wen kontaktieren:**
- HR-Leads, Support-Teamleiter, Ops-Manager in 10-100 Personen Unternehmen
- Suche in: LinkedIn, relevante Slack-Communities, Reddit r/SaaS, IndieHackers

**Wo posten:**
- HackerNews (Show HN)
- ProductHunt
- Relevante Subreddits
- LinkedIn mit konkretem Use-Case-Beispiel

**Was fragen:**
- "Wie löst du aktuell [Problem]?"
- "Was stört dich am meisten an deinem aktuellen Workflow?"
- "Würdest du X€/Monat zahlen, wenn [Lösung]?"

## Risiken

- Slack/Discord ändern Bot-APIs
- Intercom/Zendesk integrieren eigene KI
- Bot-Fatigue bei Nutzern
